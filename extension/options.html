OpenNIT Vault

Server-Verbindung

Adresse Ihrer OpenNIT-Instanz – ohne abschließenden Slash.
Empfohlen: Anmeldung wie an OpenNIT (lokal + 2FA / Microsoft 365 / Keycloak). Der Zugang wird automatisch erneuert.
Erweitert: manueller Token
Alternative ohne SSO (z. B. Kiosk/Headless): Vault → „Extension“ → Token generieren.

Sicherheit

Ist im Web-Tresor ein Tresor-PIN aktiv, verlangt die Erweiterung diesen PIN – serverseitig erzwungen, damit ein gestohlener Token allein nichts nützt. Diese Einstellung legt nur fest, wie lange eine Entsperrung gilt. Ohne gesetzten Tresor-PIN entfällt die Sperre.
Kopierte Passwörter und 2FA-Codes werden nach kurzer Zeit automatisch aus der Zwischenablage entfernt.

Einrichtungsschritte

  • 1Öffnen Sie den Passwort-Vault in Ihrer OpenNIT-Anwendung.
  • 2Klicken Sie auf „Extension“ in der Toolbar und generieren Sie einen API-Token.
  • 3Fügen Sie den Token oben in das Token-Feld ein und klicken Sie auf Speichern.
  • 4Fokussieren Sie ein Login-Feld auf einer Website – Vault-Vorschläge erscheinen automatisch.
